# ATHENA Dev Session — 2026-04-27

## Overview

This session completed Phases 3 and 4 of the ATHENA M&E platform API, added native password authentication, answered database/migration questions, and added a unified `npm run dev` command.

---

## Completed Work

### Phase 3 — Ingestion & DQA

- **`apps/api/src/routes/quarantine.ts`** — Created. Endpoints:
  - `GET /quarantine` — paginated, filterable quarantine queue
  - `POST /quarantine/:id/resolve` — resolves a quarantined submission, writes audit log
  - `POST /quarantine/:id/reject` — rejects with note, writes audit log
- **`apps/api/src/routes/conflicts.ts`** — Created. Endpoints:
  - `GET /conflicts` — list open conflicts
  - `GET /conflicts/:id` — detail with canonical payload
  - `POST /conflicts/:id/resolve` — accept_branch / keep_canonical / manual_merge, updates sector table, writes audit log
- **`apps/api/src/modules/dqa/pipeline.ts`** — `assertSafeIdentifier` exported (was private; needed by later modules)
- **`apps/api/src/app.ts`** — quarantine and conflicts routes registered

### Phase 4 — Reporting & Indicators

- **`apps/api/src/routes/indicators.ts`** — Created. Full CRUD for ITT indicators (admin/supervisor read, admin write). `POST /indicators/:id/compute` runs COUNT/SUM/AVG/COUNT_DISTINCT against the sector table and upserts into `indicator_actuals`.
- **`apps/api/src/routes/reporting.ts`** — Created. Endpoints:
  - `GET /reporting/submissions` — paginated, filterable
  - `GET /reporting/submissions/:id` — detail with entity + enumerator metadata
  - `GET /reporting/entities/:id/timeline` — all submissions across all forms for one entity
  - `GET /reporting/map` — GeoJSON FeatureCollection
  - `GET /reporting/summary` — ITT snapshot with `progress_pct`
- **`apps/api/src/app.ts`** — indicators and reporting routes registered

### Native Password Authentication

- **Migration**: `apps/database/migrations/1745750400000_add_password_auth.sql` — adds `password_hash TEXT`, `failed_login_attempts INTEGER DEFAULT 0`, `locked_until TIMESTAMPTZ` to `public.users`. Applied to DB.
- **`apps/api/src/routes/auth.ts`** — Added:
  - `POST /auth/login` — rate-limited (express-rate-limit), bcrypt compare, 5-strike lockout with 15-min freeze
  - `POST /auth/change-password` — requireAuth, validates current password, hashes new password (cost 12)
- **`apps/api/src/routes/org.ts`** — `InviteSchema` extended with optional `password` (min 12 chars). `POST /org/users` hashes password if provided and sets `is_active=TRUE`. New `POST /org/users/:id/set-password` (admin only, resets lockout, writes audit log).
- **Dependencies added**: `bcryptjs@^3.0.3`, `express-rate-limit@^8.4.1` (runtime); `@types/bcryptjs`, `@types/express-rate-limit` (dev).

### Bootstrap Seed

- **`apps/database/seeds/start-seed.sql`** — created by user (visible in session). Inserts one organisation (`Save the Children`) and one admin user (`admin@athena.local`) with a bcrypt password hash. Must be run after migrations: `psql $DATABASE_URL -f seeds/bootstrap.sql`.

### Unified Dev Command (`concurrently`)

- Installed `concurrently` as a root dev dependency.
- Root `package.json` updated — new script:
  ```json
  "dev": "concurrently --names \"api,web\" --prefix-colors \"cyan,magenta\" \"npm run dev:api\" \"npm run dev:web\""
  ```
- `npm run dev` from repo root now starts both API (nodemon, port 3000) and web (Vite, port 5173) with colour-coded output. Ctrl+C kills both.
